# JARVIS PC V2 Minimal Assistant

## Главный запуск (Рекомендуется)
Запустите **`START_JARVIS.bat`** в корне проекта.
Это единственный и универсальный файл запуска для разработчика. Он автоматически:
- Очищает старые зависшие процессы и освобождает порт 8000;
- Выполняет `git pull` для получения свежих обновлений;
- Проверяет и устанавливает необходимые зависимости бэкенда и фронтенда;
- Запускает FastAPI Backend из исходников и ожидает его готовности;
- Запускает Vite Frontend Server;
- Запускает Electron в режиме разработки;
- Чисто и безопасно завершает работу бэкенда и фронтенда при выходе из Electron.

## Старые файлы запуска (Legacy)
- **`RUN_DEV_FRESH.bat`** — *Legacy dev launch*, автоматически перенаправляет на `START_JARVIS.bat`.
- **`UPDATE_AND_LAUNCH_APP.bat`** — *Legacy production build*, используется только для сборки дистрибутива, не для обычного повседневного запуска.


## Где лежит актуальное приложение
`app_current/JARVIS PC V2.exe` (это финальная рабочая папка, откуда всегда можно запускать свежую версию).

## Почему старый exe не обновляется
Потому что Electron package содержит уже собранный frontend/backend. После изменений кода надо пересобрать приложение или запускать dev mode.

## Как очистить старые архивные сборки
Скрипт не удаляет файлы безвозвратно. Он переносит старые сборки, установщики и legacy-артефакты в `_archive/old_builds/YYYY-MM-DD_HH-MM/`.
Для управления архивами можно запустить:
```powershell
python tools\cleanup_old_builds.py
```
После запуска проверьте отчеты:
- `tools/cleanup_report.txt`
- `tools/cleanup_report.json`
